Ты работаешь с полноценным репозиторием проекта Watch Together.

Твоя задача — провести полный аудит проекта, устранить существующие ошибки, оптимизировать архитектуру, стабилизировать голосовую связь и демонстрацию экрана, создать полноценное Windows desktop-приложение для хоста, постоянный веб-клиент для гостей и реализовать безопасное совместное управление экраном хоста.

Не ограничивайся точечным исправлением первого найденного бага. Изучи проект целиком, исправляй системные причины ошибок, добавляй тесты и не называй задачу завершённой, пока основные пользовательские сценарии реально не проверены.

# Конечный результат

Проект должен работать следующим образом:

- Хост запускает обычное Windows-приложение Watch Together через .exe.
- Хосту не нужно открывать браузер, localhost, BAT-файлы или терминал.
- Desktop-приложение подключается к постоянному удалённому серверу.
- Хост нажимает «Создать комнату».
- Приложение выдаёт нормальную постоянную HTTPS-ссылку вида:
```text
https://watch.example.com/room/ROOM_ID
```

- Хост отправляет эту ссылку другу.
- Друг открывает ссылку в Chrome или Edge без установки программы.
- Друг видит демонстрацию экрана, слышит системный звук и голос хоста.
- Участники могут общаться голосом и в чате.
- Участники могут рисовать поверх демонстрации экрана.
- Гость может запросить управление мышью и клавиатурой компьютера хоста.
- Хост явно подтверждает или отклоняет запрос.
- Во время управления у хоста постоянно отображается заметный индикатор.
- Хост может мгновенно остановить управление кнопкой или аварийной горячей клавишей.
- При разрыве соединения управление автоматически прекращается.
- После завершения комнаты все медиапотоки, соединения, таймеры и разрешения очищаются.
localhost, временные туннели и BAT-файлы разрешены только в development-режиме. Production-версия не должна от них зависеть.

# Этап 12. Совместное управление экраном

Реализуй полноценное управление компьютером хоста через desktop-host.

Веб-браузер гостя только отправляет разрешённые события.

Реальное выполнение событий мыши и клавиатуры выполняется только desktop-приложением хоста.

## Сценарий

Гость нажимает «Запросить управление».

Хост получает заметное окно с:

- именем участника;
- разрешениями;
- кнопкой «Разрешить»;
- кнопкой «Отклонить».
Хост выбирает разрешения:

- только мышь;
- мышь и клавиатура.
После подтверждения создаётся control session.

У хоста появляется постоянный заметный индикатор.

Гость видит, что управление разрешено.

Хост может в любой момент:

- нажать «Остановить»;
- использовать аварийную горячую клавишу;
- закрыть комнату;
- отозвать только клавиатуру;
- полностью отозвать управление.
При disconnect управление автоматически прекращается.

## Обязательные ограничения безопасности

Удалённое управление запрещено без явного подтверждения хоста.

Не реализовывай:

- скрытое управление;
- автоматическое подтверждение;
- запуск без видимого индикатора;
- сохранение разрешения навсегда;
- управление после reconnect без нового подтверждения;
- запись введённого текста;
- keylogging;
- clipboard stealing;
- доступ к файлам без отдельной функции;
- управление до входа в комнату;
- управление другим компьютером, кроме текущего desktop-host;
- выполнение произвольных shell-команд;
- удалённый запуск PowerShell или CMD;
- обход Windows permissions;
- повышение привилегий;
- управление экраном блокировки;
- скрытие курсора хоста;
- маскировку активной сессии.
## Control session

Каждая сессия должна иметь:

- уникальный session ID;
- host ID;
- controller ID;
- разрешённые capabilities;
- timestamp начала;
- expiration;
- heartbeat;
- sequence number;
- nonce;
- состояние active/revoked/expired;
- ограничение частоты событий.
События без правильного session ID должны игнорироваться.

События после revoke должны игнорироваться.

События со старым sequence number должны игнорироваться.

После reconnect требуется новое подтверждение.

Координаты должны передаваться нормализованно:

```text

```

- разные aspect ratio;
- letterboxing;
- fullscreen;
- смену выбранного монитора;
- изменение stream resolution.
Не допускай выхода координат за границы разрешённой области.

При демонстрации конкретного окна управление должно быть ограничено этим окном, если технология это позволяет.

Если безопасное ограничение конкретным окном невозможно, явно предупреди хоста перед выдачей разрешения.

Обязательно обрабатывай:

- keyup;
- modifier keys;
- потерю фокуса;
- stuck keys;
- disconnect;
- revoke;
- emergency stop.
После остановки управления desktop-host должен принудительно отпустить все виртуально зажатые клавиши и кнопки мыши.

Опасные системные сочетания должны быть заблокированы или требовать отдельного разрешения.

Не отправляй control events, когда пользователь печатает в:

- input;
- textarea;
- contenteditable;
- чате;
- настройках.
